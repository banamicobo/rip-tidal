# Sesja 2026-05-16 — instalacja streamrip + Tidal

**TL;DR**: Po długich zmaganiach zainstalowany i działający streamrip do pobierania muzyki z Tidal. Dwa patche w source code streamrip, autoryzacja przez tidalapi.

---

## Co zrobione

- Zainstalowany Python 3.12 (3.14 za nowy dla zależności Pillow)
- Zainstalowany `streamrip` 2.1.0 i `tidalapi` przez `py -3.12 -m pip install`
- Dodany `Scripts` folder Python 3.12 do PATH
- Autoryzacja Tidal przez `tidalapi.Session().login_oauth_simple()`
- Tokeny wklejone do `C:\Users\banam\AppData\Roaming\streamrip\config.toml`
- **Patch #1**: podmiana CLIENT_ID/SECRET w `streamrip/client/tidal.py` na kredencjale tidalapi (zablokowane własne klucze streamrip)
- **Patch #2**: `except TypeError` → `except Exception` przy fetchowaniu lyrics (401 crashował cały download)
- Repo `rip-tidal` stworzone na GitHubie z pełną dokumentacją

## Co nie działało (i dlaczego)

- `tidal-dl` — stare API z 2022, Tidal blokuje jego klucze (403)
- `streamrip` na Python 3.14 — Pillow nie ma wheel dla 3.14
- Device login streamrip/tidal-dl — Tidal blokuje ich client_id przy token exchange
- Lyrics endpoint (`/lyrics`) zwraca 401 — inny host niż API, token nie przechodzi
